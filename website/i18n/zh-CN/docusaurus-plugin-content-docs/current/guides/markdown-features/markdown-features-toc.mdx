# 标题和目录

## Markdown 标题 {#markdown-headings}

您可以使用常规的 Markdown 标题。

```md
## 2 级标题

### 3 级标题

#### 4 级标题
```

每个 Markdown 标题都会作为目录条目出现。

### 标题 ID {#heading-ids}

每个标题都有一个 ID，可以自动生成或明确指定。标题 ID 允许您在 Markdown 或 JSX 中链接到特定的文档标题：

```md
[链接](#heading-id)
```

```jsx
<Link to="#heading-id">链接</Link>
```

默认情况下，Docusaurus 会根据标题文本为您生成标题 ID。例如，`### 你好世界` 的 ID 将是 `hello-world`。

生成的 ID 有**一些限制**：

- ID 可能看起来不好看
- 您可能希望**更改或翻译**文本而不更新现有的 ID

一种特殊的 Markdown 语法让您可以设置一个**明确的标题 ID**：

```mdx-code-block
<Code language="md">{'### 你好世界 \u007B#my-explicit-id}\n'}</Code>
```

:::tip

使用 **[`write-heading-ids`](../../cli.mdx#docusaurus-write-heading-ids-sitedir)** CLI 命令为您的所有 Markdown 文档添加明确的 ID。

:::

:::warning 避免 ID 冲突

生成的标题 ID 将保证在每个页面上是唯一的，但如果您使用自定义 ID，请确保每个 ID 在每个页面上只出现一次，否则会有两个 DOM 元素具有相同的 ID，这是无效的 HTML 语义，并将导致一个标题无法链接。

:::

## 目录标题级别 {#table-of-contents-heading-level}

每个 Markdown 文档在右上角显示一个目录。默认情况下，此表格仅显示 h2 和 h3 标题，这对于页面结构的概述应该足够了。如果您需要更改显示的标题范围，您可以自定义最小和最大标题级别——可以按页面或全局进行。

要为特定页面设置标题级别，请使用 `toc_min_heading_level` 和 `toc_max_heading_level` front matter。

```md title="myDoc.md"
---
# 显示 h2 到 h5 标题
toc_min_heading_level: 2
toc_max_heading_level: 5
---
```

要为_所有_页面设置标题级别，请使用 [`themeConfig.tableOfContents`](../../api/themes/theme-configuration.mdx#table-of-contents) 选项。

```js title="docusaurus.config.js"
export default {
  themeConfig: {
    tableOfContents: {
      // highlight-start
      minHeadingLevel: 2,
      maxHeadingLevel: 5,
      // highlight-end
    },
  },
};
```

如果您已全局设置选项，您仍然可以通过 front matter 在本地覆盖它们。

:::note

`themeConfig` 选项将应用于站点上的所有目录，包括[内联目录](#inline-table-of-contents)，但 front matter 选项仅影响右上角的目录。您需要使用 `minHeadingLevel` 和 `maxHeadingLevel` props 来自定义每个 `<TOCInline />` 组件。

:::

## 内联目录 {#inline-table-of-contents}

借助 MDX，也可以在 Markdown 文档中直接显示内联目录。

`toc` 变量在任何 MDX 文档中都可用，并包含 MDX 文档的所有标题。默认情况下，目录中只显示 `h2` 和 `h3` 标题。您可以通过为单个 `TOCInline` 组件设置 `minHeadingLevel` 或 `maxHeadingLevel` 来更改可见的标题级别。

{/* prettier-ignore */}
```jsx
import TOCInline from '@theme/TOCInline';

<TOCInline toc={toc} />
```

```mdx-code-block
import TOCInline from '@theme/TOCInline';

<BrowserWindow>
<TOCInline toc={toc} />
</BrowserWindow>
```

`toc` 全局变量只是一个标题项目列表：

```ts
declare const toc: {
  value: string;
  id: string;
  level: number;
}[];
```

请注意，`toc` 全局变量是一个扁平数组，因此您可以轻松地剪切掉不需要的节点或插入额外的节点，并创建一个新的目录树。

{/* prettier-ignore */}
```jsx
import TOCInline from '@theme/TOCInline';

<TOCInline
  // 只显示 h2 和 h4 标题
  toc={toc.filter((node) => node.level === 2 || node.level === 4)}
  minHeadingLevel={2}
  // 除默认的 h2 和 h3 标题外，还显示 h4 标题
  maxHeadingLevel={4}
/>
```

```mdx-code-block
<BrowserWindow>
<TOCInline
  toc={toc.filter((node) => node.level === 2 || node.level === 4)}
  minHeadingLevel={2}
  maxHeadingLevel={4}
/>
</BrowserWindow>
```

## 自定义目录生成 {#customizing-table-of-contents-generation}

目录是通过使用 [Remark 插件](./markdown-features-plugins.mdx)解析 Markdown 源生成的。在已知的边缘情况下，它会产生假阳性和假阴性。

可隐藏区域内的 Markdown 标题仍会出现在目录中。例如，[`Tabs`](./markdown-features-tabs.mdx) 和 [`details`](./markdown-features-intro.mdx#details) 中的标题不会被排除。

非 Markdown 标题不会出现在目录中。您可以利用这一点来解决上述问题。

```md
<details>
<summary>一些包含标题的详细信息</summary>
<h2 id="#heading-id">我是一个不会出现在目录中的标题</h2>

一些内容...

</details>
```

符合人体工程学地插入额外标题或忽略某些标题的功能正在开发中。如果此功能对您很重要，请在[此问题](https://github.com/facebook/docusaurus/issues/6201)中报告您的用例。
