# 文档多实例

`@docusaurus/plugin-content-docs` 插件可以支持[多实例](../../using-plugins.mdx#multi-instance-plugins-and-plugin-ids)。

:::note

此功能仅对[版本化文档](./versioning.mdx)有用。建议在阅读本页之前熟悉文档版本控制。如果您只想拥有[多个侧边栏](./sidebar/multiple-sidebars.mdx)，您可以在一个插件内完成。

:::

## 用例 {#use-cases}

有时您希望一个 Docusaurus 站点托管 2 套或更多不同的文档。

这些文档甚至可能有不同的版本控制/发布生命周期。

### 移动 SDK 文档 {#mobile-sdks-documentation}

如果您构建一个跨平台的移动 SDK，您可能会有 2 套文档：

- Android SDK 文档 (`v1.0`, `v1.1`)
- iOS SDK 文档 (`v1.0`, `v2.0`)

在这种情况下，您可以为每个移动 SDK 文档使用一个不同的文档插件实例。

:::warning

如果每个文档实例都非常大，您最好创建 2 个不同的 Docusaurus 站点。

如果有人编辑了 iOS 文档，重新构建所有内容，包括整个未更改的 Android 文档，真的有用吗？

:::

### 版本化和非版本化文档 {#versioned-and-unversioned-doc}

有时，您希望某些文档是版本化的，而其他文档则更“全局化”，对它们进行版本控制感觉毫无用处。

我们在 Docusaurus 网站本身就使用了这种模式：

- [/docs/\*](/docs) 部分是版本化的
- [/community/\*](/community/support) 部分是非版本化的

## 设置 {#setup}

假设您有 2 套文档：

- 产品：一些关于您产品的版本化文档
- 社区：一些关于您产品周围社区的非版本化文档

在这种情况下，您应该在您的站点配置中两次使用同一个插件。

:::warning

`@docusaurus/preset-classic` 已经为您包含了一个文档插件实例！

:::

使用 preset 时：

```js title="docusaurus.config.js"
export default {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        docs: {
          // highlight-start
          // id: 'product', // 省略 => 默认实例
          // highlight-end
          path: 'product',
          routeBasePath: 'product',
          sidebarPath: './sidebarsProduct.js',
          // ... 其他选项
        },
      },
    ],
  ],
  plugins: [
    [
      '@docusaurus/plugin-content-docs',
      {
        // highlight-start
        id: 'community',
        // highlight-end
        path: 'community',
        routeBasePath: 'community',
        sidebarPath: './sidebarsCommunity.js',
        // ... 其他选项
      },
    ],
  ],
};
```

不使用 preset 时：

```js title="docusaurus.config.js"
export default {
  plugins: [
    [
      '@docusaurus/plugin-content-docs',
      {
        // highlight-start
        // id: 'product', // 省略 => 默认实例
        // highlight-end
        path: 'product',
        routeBasePath: 'product',
        sidebarPath: './sidebarsProduct.js',
        // ... 其他选项
      },
    ],
    [
      '@docusaurus/plugin-content-docs',
      {
        // highlight-start
        id: 'community',
        // highlight-end
        path: 'community',
        routeBasePath: 'community',
        sidebarPath: './sidebarsCommunity.js',
        // ... 其他选项
      },
    ],
  ],
};
```

不要忘记为插件实例分配一个唯一的 `id` 属性。

:::note

我们认为 `product` 实例是最重要的，因此通过不分配任何 ID 使其成为“默认”实例。

:::

## 版本化路径 {#versioned-paths}

每个插件实例都会将版本化的文档存储在一个不同的文件夹中。

默认插件实例将使用以下路径：

- `website/versions.json`
- `website/versioned_docs`
- `website/versioned_sidebars`

其他插件实例（带有 `id` 属性）将使用以下路径：

- `website/[pluginId]_versions.json`
- `website/[pluginId]_versioned_docs`
- `website/[pluginId]_versioned_sidebars`

:::tip

您可以省略其中一个文档插件实例的 `id` 属性（默认为 `default`）。

实例路径将更简单，并且与单实例设置向后兼容。

:::

## 标记新版本 {#tagging-new-versions}

每个插件实例都有自己的 CLI 命令来标记新版本。如果您运行以下命令，它们将被显示：

```bash npm2yarn
npm run docusaurus -- --help
```

要对产品/默认文档插件实例进行版本控制：

```bash npm2yarn
npm run docusaurus docs:version 1.0.0
```

要对非默认/社区文档插件实例进行版本控制：

```bash npm2yarn
npm run docusaurus docs:version:community 1.0.0
```

## 文档导航栏项目 {#docs-navbar-items}

每个与文档相关的[主题导航栏项目](../../api/themes/theme-configuration.mdx#navbar)都有一个可选的 `docsPluginId` 属性。

例如，如果您想为每个移动 SDK（iOS 和 Android）都有一个版本下拉菜单，您可以这样做：

```js title="docusaurus.config.js"
export default {
  themeConfig: {
    navbar: {
      items: [
        {
          type: 'docsVersionDropdown',
          // highlight-start
          docsPluginId: 'ios',
          // highlight-end
        },
        {
          type: 'docsVersionDropdown',
          // highlight-start
          docsPluginId: 'android',
          // highlight-end
        },
      ],
    },
  },
};
```
