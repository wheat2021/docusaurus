# 接下来做什么？

恭喜！您现在已经了解了 Docusaurus 的大部分核心功能。您已经：

- [使用页面插件](./creating-pages.mdx)创建了一个独立的 React / Markdown 页面
- [使用文档插件](./docs/docs-introduction.mdx)创建了文档页面。这包括[配置侧边栏](./docs/sidebar/index.mdx)，甚至[版本控制](./docs/versioning.mdx)
- [使用博客插件](../blog.mdx)创建了一个功能齐全的博客
- 尝试了[一系列 Markdown 功能](./markdown-features/markdown-features-intro.mdx)，这些功能对所有内容插件都很有用
- [使用样式表](../styling-layout.mdx)或 [swizzling](../swizzling.mdx) 来自定义您网站的外观
- 在您的页面中[包含了图片和其他资源](../static-assets.mdx)
- 为您的网站[添加了搜索功能](../search.mdx)
- 理解了如何通过标准的 Docusaurus API 来完成[浏览器支持](../browser-support.mdx)和 [SEO](../seo.mdx)
- 学习了如何安装和配置[单个插件](../using-plugins.mdx)
- 将您的网站[部署](../deployment.mdx)到内容托管服务商
- 将您的网站[国际化](../i18n/i18n-tutorial.mdx)以包含多种语言

此时，您可能已经有了一个很大的 `docusaurus.config.js` 文件😄 然而，您还没有编写太多代码！大多数功能都是通过调用封装好的 Docusaurus API 来实现的。在您继续您的旅程时，您可以选择三条路径：

- 学习更高级的 Docusaurus 概念。这将帮助您更深入地理解这些 API 的作用。
- 阅读关于[所有现有 API](../docusaurus-core.mdx) 的文档。其中许多在指南中尚未涵盖！
- 学习如何[开发一个插件](../api/plugin-methods/README.mdx)来扩展您网站的功能。
