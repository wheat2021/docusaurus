# 📦 create-docusaurus

一个脚手架工具，可帮助你立即搭建一个功能齐全的 Docusaurus 应用。

## 用法 {#usage}

```bash
npx create-docusaurus@latest [name] [template] [rootDir]
```

`name` 参数将用作网站的路径，以及所创建应用 package.json 中的 `name` 字段。它可以是绝对路径，也可以是相对于 `rootDir` 的路径。

`template` 参数可以是以下之一：

- `classic`: 使用经典模板（推荐）
- `facebook`: 使用 Facebook/Meta 模板，其中包含一些 Meta 特定的设置
- 一个 git 仓库 URL（以 `https://` 或 `git@` 开头），可以克隆到目标位置
- 一个相对于当前工作目录（CWD）的本地文件路径，其中包含要复制到目标位置的文件

`rootDir` 将用于解析网站目录的绝对路径。默认为当前工作目录（CWD）。

:::warning

此命令最好在交互式 shell 中使用，以便所有功能都可用。

:::

## 选项 {#options}

### `-t, --typescript` {#typescript}

当模板参数是可识别的名称时使用。目前，只有 `classic` 提供了 TypeScript 版本。

### `-g, --git-strategy` {#git-strategy}

当模板参数是 git 仓库时使用。它需要是以下之一：

- `deep`: 保留完整的 git 历史记录
- `shallow`: 使用 `--depth=1` 克隆
- `copy`: 进行浅克隆，但不创建 git 仓库
- `custom`: 输入你的自定义 git 克隆命令。我们会提示你输入。你可以写一些类似 `git clone --depth 10` 的命令，我们会附加仓库 URL 和目标目录。

### `-p, --package-manager` {#package-manager}

值应为 `npm`、`yarn`、`pnpm` 或 `bun` 之一。如果未明确提供，Docusaurus 将根据以下情况推断一个：

- 当前工作目录（CWD）中已存在的锁文件（例如，如果你在现有项目中设置网站）
- 用于调用 `create-docusaurus` 的命令（例如 `npm init`、`npx`、`yarn create`、`bunx` 等）
- 如果没有所有启发式方法，则进行交互式提示

### `-s, --skip-install` {#skip-install}

如果提供，Docusaurus 在创建应用后不会自动安装依赖项。`--package-manager` 选项仅在你实际安装依赖项时有用。
