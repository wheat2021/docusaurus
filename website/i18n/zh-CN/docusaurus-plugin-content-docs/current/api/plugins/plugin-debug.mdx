# 📦 plugin-debug

```mdx-code-block
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
```

调试插件将在 [`http://localhost:3000/__docusaurus/debug`](http://localhost:3000/__docusaurus/debug) 显示有用的调试信息。

它主要对插件作者有用，他们将能够更轻松地检查 `.docusaurus` 文件夹的内容（如创建的路由），也能够检查从未写入磁盘的数据结构，如通过 `contentLoaded` 生命周期加载的插件数据。

:::info

如果你通过经典预设使用此插件，预设将默认**在开发中启用插件，在生产中禁用**（`debug: undefined`），以避免暴露潜在的敏感信息。你可以使用 `debug: true` 始终启用它，或使用 `debug: false` 始终禁用它。

如果你使用独立插件，你可能需要通过检查环境来达到同样的效果：

```js title="docusaurus.config.js"
export default {
  plugins: [
    // highlight-next-line
    process.env.NODE_ENV !== 'production' && '@docusaurus/plugin-debug',
  ].filter(Boolean),
};
```

:::

:::note

如果你报告一个 bug，我们很可能会要求你在生产中打开这个插件，以便我们能更轻松地检查你的部署配置。

如果你没有任何敏感信息，你可以在生产中保持开启状态，[就像我们一样](/__docusaurus/debug)。

:::

## 安装 {#installation}

```bash npm2yarn
npm install --save @docusaurus/plugin-debug
```

:::tip

如果你使用预设 `@docusaurus/preset-classic`，则无需将此插件作为依赖项安装。

你可以通过[预设选项](../../using-plugins.mdx#docusauruspreset-classic)配置此插件。

:::

## 配置 {#configuration}

此插件目前没有选项。

### 示例配置 {#ex-config}

你可以通过预设选项或插件选项来配置此插件。

:::tip

大多数 Docusaurus 用户通过预设选项配置此插件。

:::

```mdx-code-block
<Tabs groupId="api-config-ex">
<TabItem value="preset" label="预设选项">
```

如果你使用预设，请通过[预设选项](../../using-plugins.mdx#docusauruspreset-classic)配置此插件：

```js title="docusaurus.config.js"
export default {
  presets: [
    [
      '@docusaurus/preset-classic',
      {
        // highlight-next-line
        debug: true, // 这将在生产中启用插件
      },
    ],
  ],
};
```

```mdx-code-block
</TabItem>
<TabItem value="plugin" label="插件选项">
```

如果你正在使用独立插件，请直接向插件提供选项：

```js title="docusaurus.config.js"
export default {
  // highlight-next-line
  plugins: ['@docusaurus/plugin-debug'],
};
```

</TabItem>
</Tabs>
